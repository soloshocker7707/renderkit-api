1. Screenshot Capture

Endpoint: POST /v1/screenshot/capture. Tests validate standard, clean, and debug modes across diverse web environments. All tests used wait: "smart" for DOM stability detection.

1.1 CNN — Heavy JS Stabilization

https://www.cnn.com

Configuration: wait: "smart", stealth: true, quality: 70

CNN is one of the heaviest JavaScript environments on the web. RenderKit's Smart Wait Engine successfully analyzed DOM stability over time and captured only after the page fully settled. The auto-clean engine reduced render time by 47% from 19.4s to 10.3s by removing intrusive overlays and ads.

1.2 YCombinator — Navigation Resilience

https://news.ycombinator.com

Configuration: wait: "smart", stealth: true, quality: 70

YCombinator's HN experienced a navigation timeout on first load (standard mode). RenderKit's graceful degradation handled this by proceeding with capture anyway, resulting in a long render time. The clean mode avoided the timeout entirely and completed in 11.7s. This demonstrates the resiliency built into the capture pipeline.

7. Cache System
