API Docs

Browser rendering infrastructure for screenshots, PDFs, and OG images.

RenderKit API gives teams a reliable headless browser rendering layer for automation workflows, reporting systems, marketing previews, and developer tooling. Send a URL, HTML document, or template payload and receive production-ready image or PDF output.

Base URLhttps://api.renderkit.com
Auth Headerx-api-key
Versionv1
FormatsPNG, JPEG, PDF
Use the public gateway URL in production. Direct backend access is protected and intended only for internal gateway forwarding.

Authentication

All rendering endpoints require an API key. Include the key in the x-api-key request header. The health endpoint is public and does not require authentication.

Header
x-api-key: YOUR_API_KEY

Quickstart

The capture endpoint accepts JSON and returns a base64-encoded image by default. For binary output, send an image-specific Accept header.

cURLScreenshot request 
curl -X POST https://api.renderkit.com/v1/screenshot/capture \
  -H "Content-Type: application/json" \
  -H "x-api-key: YOUR_API_KEY" \
  -d '{
    "url": "https://example.com",
    "width": 1280,
    "height": 800,
    "format": "png",
    "wait": "smart",
    "clean": true
  }'

Endpoints

POST/v1/screenshot/capture

Capture Screenshot

Captures a URL, raw HTML document, or named template as a PNG or JPEG image. Use this endpoint for website snapshots, visual previews, QA automation, and dynamic image generation.

Request body 
{
  "url": "https://example.com",
  "width": 1280,
  "height": 800,
  "fullPage": false,
  "format": "jpeg",
  "quality": 70,
  "wait": "smart",
  "clean": true,
  "freezeAnimations": true,
  "debug": false
}
POST/v1/screenshot/pdf

Generate PDF

Converts URLs, raw HTML, or templates into PDF documents. Common use cases include reports, invoices, statements, confirmations, receipts, and internal document automation.

Request body 
{
  "html": "<html><body><h1>Report</h1></body></html>",
  "format": "A4",
  "landscape": false,
  "printBackground": true,
  "clean": false,
  "debug": false
}
POST/v1/screenshot/og

Generate Open Graph Image

Creates a 1200 by 630 PNG social preview image from structured title, description, theme, background color, and optional logo input.

Request body 
{
  "title": "Launch faster with RenderKit",
  "description": "Screenshots, PDFs, and OG images from one API.",
  "theme": "dark",
  "background_color": "#000000",
  "logo_url": "https://example.com/logo.png"
}
GET/health

Health Check

Returns service status and version metadata. This endpoint is public and does not consume authenticated render quota.

Response
{
  "status": "ok",
  "version": "1.1.0 (Modular)"
}

Parameters

Most rendering options are optional. Provide one input source: url, html, or template.

Name Type Endpoints Description
url string Screenshot, PDF Public page URL to render.
html string Screenshot, PDF Raw HTML document to render directly.
template string Screenshot, PDF Name of a server-side HTML template.
data object Templates Values used to replace template placeholders.
width, height number Screenshot, PDF Viewport dimensions before rendering.
format string All render endpoints Image format for screenshots or paper format for PDFs.
wait string Screenshot Use smart for stability detection or Puppeteer wait modes.
clean boolean Screenshot, PDF Hide common popups, consent banners, and intrusive overlays.
css string Screenshot, PDF Custom CSS injected before capture.
headers object URL rendering Additional HTTP headers sent during navigation.
debug boolean All render endpoints Includes render timings and processing actions.
noCache boolean All render endpoints Bypasses cached output for the request.

Responses

JSON response

Default responses include success, base64 content, render timing, request ID, and timestamp. Screenshots and OG images return image_base64. PDFs return pdf_base64.

Binary response

Send Accept: image/png, Accept: image/jpeg, or Accept: application/pdf to receive the rendered file directly.

Example JSON response 
{
  "success": true,
  "image_base64": "...",
  "format": "png",
  "render_time": 4225,
  "request_id": "abc123",
  "timestamp": "2026-05-29T00:00:00.000Z"
}

Errors

Status Code Meaning
400 validation_error Required input is missing or invalid.
401 DIRECT_ACCESS_FORBIDDEN Request is missing gateway authorization or API authentication.
404 template_not_found The requested template does not exist.
500 render_failed The browser renderer could not complete the request.

Limits and caching

RenderKit uses gateway-level rate limits and monthly quotas based on your plan. Equivalent render requests may be cached to reduce latency and browser usage. Set noCache: true for fresh output.

Smart stabilization

Smart wait observes font readiness, lazy loading, and DOM stability before capture to improve output consistency for SPAs and dynamic pages.

Gateway-first security

Production requests should use the public API gateway. The backend expects an internal gateway secret and rejects direct unauthenticated traffic.
